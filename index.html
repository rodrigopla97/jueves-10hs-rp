Areas de GIT

Working directory

Es la carpeta donde realizamos los cambios en nuestro proyecto, en este lugar rara vez utilizaremos la consola y es el área donde se visualizan todos los cambios de los archivos

Staging Area (Index)

Es el área de control de cambios, donde se verfucan las modificaciones realizadas en los archivos del proyecto, Aquí se realizan los snapshots de codigo para luego ser confirmados.

Repository (Local / Remote)

Es el área donde se almacena las capturas en formato BLOB (Binary Object File) a las cuales podemos acceder mediante el historial de cambios. Para quenerar estos archivos debemos emitir un commit, el mismo valida los cambios y los almacena en la base de datos de GIT.
